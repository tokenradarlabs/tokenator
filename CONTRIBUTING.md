## Getting Started

1. **Fork the repository** and clone it to your local machine.
2. Install dependencies:
   ```bash
   npm install
   # or
   yarn install
   ```
3. Start the development server:
   ```bash
   npm run dev
   # or
   yarn dev
   ```
4. Make your changes in a new branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Code Style

- Use [Prettier](https://prettier.io/) and [ESLint](https://eslint.org/) for code formatting and linting.
- Follow the existing code style in the project.
- Use TypeScript for all files.
- Use [Tailwind CSS](https://tailwindcss.com/) utility classes for styling.
- Keep components small and focused.
- Place reusable UI components in `src/components/ui/` and common sections in `src/components/common/`.

## Commit Messages

- Use clear, descriptive commit messages.
- Follow the [Conventional Commits](https://www.conventionalcommits.org/) style:
  - `feat: add new feature`
  - `fix: fix a bug`
  - `docs: update documentation`
  - `refactor: code refactoring`
  - `style: formatting, missing semi colons, etc.`
  - `test: add or update tests`

## Submitting Issues

- Search [existing issues](https://github.com/your-repo/issues) before opening a new one.
- Provide a clear and descriptive title and description.
- Include steps to reproduce, expected behavior, and screenshots if applicable.

## Pull Requests

- Ensure your branch is up to date with `main` before submitting a PR.
- Provide a clear description of your changes.
- Reference related issues in your PR description (e.g., `Closes #123`).
- Ensure all checks pass (lint, build, tests).
- Request a review from a maintainer.
